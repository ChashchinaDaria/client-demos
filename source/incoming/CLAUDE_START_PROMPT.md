# ЗАПУСК ОБРАБОТКИ КЛИЕНТОВ

Используй прикрепленный файл `Claude_clients_Russia_verified.xlsx` как единственный источник фактов о компаниях.

Сначала полностью обработай клиентов с приоритетом 1–3.

Для каждого клиента:
1. Прочитай строку целиком и открой официальный сайт из поля «Источник», если веб-доступ разрешен.
2. Выбери одно основное демо и один небольшой дополнительный элемент, указанные в таблице; меняй выбор только при явном обосновании.
3. Создай папку `clients/<client-id>-<slug>/`.
4. Создай автономное интерактивное демо `demo/index.html` со встроенными HTML, CSS и JavaScript. Оно должно открываться локально, работать на мобильном и не использовать API.
5. Реализуй только один законченный сценарий. Остальные возможности вынеси в блок «В полной версии».
6. Добавь подпись: «Персональная демонстрационная концепция. Не является официальным сайтом компании».
7. Создай:
   - `analysis.md`
   - `messages/messenger.txt`
   - `messages/email.txt`
   - `messages/followup_1.txt`
   - `messages/followup_2.txt`
   - `recommended_package.md`
   - `metadata.json`
8. В первом сообщении сразу вставь `{{DEMO_URL}}`.
9. Не придумывай контакты, CRM, количество заявок, финансовые показатели, клиентов компании или точные цены ее продукции.
10. Все условные цифры внутри демо помечай «демонстрационные данные» или «предварительный расчет, не оферта».
11. Создай общие файлы:
   - `master_clients.csv`
   - `processing_summary.md`
   - `QA_REPORT.md`
12. Проверь синтаксис каждого HTML и работу всех кнопок.
13. Верни ZIP-архив `clients_ready_to_send.zip`, а не только описание работы.

Главная цель: создать небольшое персонализированное демо, которое можно сразу опубликовать и отправить потенциальному клиенту, чтобы получить ответ и назначить разговор.
